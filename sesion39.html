SSH (Secure Shell) vs. Telnet y Uso del Cliente PuTTY

1. SSH (Secure Shell): El Estándar de Consola Remota

**SSH** es el protocolo que todo técnico debe usar para acceder a la línea de comandos (CLI) de routers, switches, servidores Linux y equipos avanzados. Es el método más seguro para la administración remota.

Características

  • **Seguro:** Toda la comunicación entre cliente y servidor está **cifrada**.
  • **Puerto:** Usa el puerto estándar **TCP 22**.
  • **Autenticación:** Usa contraseñas y, a menudo, claves de cifrado (más seguras).

Telnet: El Protocolo Muerto 💀

Telnet (Puerto TCP 23) es el protocolo de consola antiguo. **NUNCA** debe usarse, ya que **envía todo el tráfico (incluyendo contraseñas) en texto plano** sin cifrar, siendo extremadamente vulnerable a la intercepción de datos.

[Image comparing SSH and Telnet security showing encryption]

2. PuTTY: El Cliente SSH Gratuito

**PuTTY** es la aplicación cliente más popular y gratuita para establecer sesiones SSH. Permite conectarse a dispositivos remotos y gestionar configuraciones desde la línea de comandos.

Tarea Práctica (Simulación/Real):

  1. Descarga e instala PuTTY (o usa el cliente SSH nativo de tu OS).
  2. Abre PuTTY y selecciona la conexión de tipo **SSH** (puerto 22).
  3. Introduce una IP local (ej. la IP de un Router configurado previamente).
  4. (Teórico): Si logras iniciar sesión, puedes empezar a configurar el router remotamente (ej. con comandos de Cisco IOS).

3. El Rol de SSH en Routers y Switches

En redes empresariales, los routers y switches deben configurarse para aceptar solo conexiones SSH. Esto implica generar claves de cifrado en el dispositivo y deshabilitar Telnet, garantizando que el acceso de administración sea seguro.
