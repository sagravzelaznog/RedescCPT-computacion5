Semana 9, Sesión 33: Puertos y NAT 🧭

Identificando Servicios y Traducción de Direcciones IP

1. Puertos de Red (Sockets): Puertas a los Servicios

Los **Puertos** son números (del 0 al 65535) que se utilizan para identificar qué aplicación o servicio debe recibir los datos en un host específico. El número de puerto se combina con la IP para formar un **Socket**.

Puertos Bien Conocidos (0 a 1023)

Reservados para servicios comunes y estándar de Internet. Ejemplos: HTTP (80), HTTPS (443), FTP (21), DNS (53).

Puertos Dinámicos/Privados (49152 a 65535)

Utilizados por los clientes (tu PC) para iniciar conexiones salientes. Son temporales y se asignan al azar.

2. NAT: El Traductor de Direcciones IP

El **NAT (Network Address Translation)** es el proceso por el cual un router modifica la dirección IP de los paquetes. Su uso más importante es permitir que **múltiples IPs privadas (LAN)** compartan una **única IP pública (WAN)** para acceder a Internet. Sin NAT, cada dispositivo necesitaría su propia IP pública, algo costoso y limitado.

IPs Privadas vs. Públicas

Las **IPs Privadas** (ej. 192.168.x.x, 10.x.x.x) solo son válidas dentro de tu red local. Las **IPs Públicas** son únicas en Internet.

Port Address Translation (PAT)

La forma más común de NAT es **PAT** (o NAPT), que usa diferentes números de puerto para distinguir qué host interno inició cada conexión saliente, permitiendo que miles de dispositivos compartan una sola IP pública.

3. NAT Estático: Reenvío de Puertos (Port Forwarding)

El **Port Forwarding** es una configuración que le dice al router: "Si alguien de Internet intenta conectarse a mi IP pública en el Puerto X, redirige ese tráfico a la IP privada Y". Esto se usa para hacer que un servidor o una consola de juegos interna sea accesible desde Internet.
