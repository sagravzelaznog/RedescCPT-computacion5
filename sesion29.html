Semana 8, Sesión 29: Control Remoto 📡

RDP, VNC y Montaje de Unidades de Red

1. Escritorio Remoto (Remote Desktop)

El **Escritorio Remoto** permite a un técnico controlar la interfaz gráfica de otro equipo a través de la red, como si estuviera sentado frente a él. Es esencial para el soporte técnico.

RDP (Remote Desktop Protocol)

Protocolo nativo de Windows (Puerto TCP 3389). Permite una conexión rápida y segura entre equipos Windows (es necesario activarlo en el equipo de destino).

VNC (Virtual Network Computing)

Un sistema de control remoto de código abierto, multiplataforma (Windows, Linux, Mac). Requiere instalar un cliente VNC tanto en el equipo de control como en el de destino.

2. Compartición de Archivos y Montaje de Unidades

Una función clave de la red es la colaboración, lo cual se logra al compartir recursos de almacenamiento (carpetas o discos duros completos).

Recursos Compartidos (Shares)

En Windows, una carpeta puede ser compartida y se accede a ella mediante la ruta **`\\[IP o Nombre]\\[Carpeta]`** (ej. `\\192.168.1.50\Documentos`). Es fundamental configurar permisos de lectura/escritura.

Montar Unidad de Red

Para facilitar el acceso continuo, un técnico puede "montar" una carpeta compartida remota y asignarle una letra de unidad local (ej. la carpeta `Documentos` del servidor se ve como la unidad **Z:** en el PC del usuario).

3. Práctica: Uso de Advanced IP Scanner para RDP

Tarea Práctica (Software Gratuito):

  1. Abre **Advanced IP Scanner**.
  2. Escanea tu red e identifica un PC activo (puede ser el tuyo con RDP habilitado).
  3. En la columna de la IP, haz clic derecho y busca las opciones de **"Control Remoto"** (RDP).
  4. Intenta iniciar una sesión RDP (necesitarás usuario y contraseña del equipo de destino).
