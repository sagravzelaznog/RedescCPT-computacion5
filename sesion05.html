Semana 2, Sesión 5: El Modelo OSI 🧠

Arquitectura de Red: El Estándar de 7 Capas

¿Qué es el Modelo OSI?

El **Modelo de Interconexión de Sistemas Abiertos (OSI)** es un marco de referencia conceptual creado por la ISO. Su propósito es estandarizar cómo los diferentes sistemas de hardware y software se comunican, dividiendo el complejo proceso de comunicación de red en siete capas distintas.

La principal ventaja es que permite a los fabricantes desarrollar productos que funcionen perfectamente juntos (interoperabilidad), sin importar su marca.

Las 7 Capas OSI (De Arriba Abajo) 🔽

El modelo se puede dividir en dos grupos: las Capas Superiores (Software, cercanas al usuario) y las Capas Inferiores (Hardware, cercanas al medio físico).

Capas de Aplicación (7, 6, 5)

Manejan los datos y la interacción del usuario.

  • 7. Aplicación: Interfaz con el usuario (navegadores, correo).
  • 6. Presentación: Formato de datos, cifrado y compresión.
  • 5. Sesión: Mantiene la conexión activa (diálogo) entre aplicaciones.

Capa de Transporte (4)

Gestiona la conexión de extremo a extremo y la confiabilidad.

  • 4. Transporte: Segmentación de datos y control de flujo. Protocolos clave: TCP y **UDP**.

Capas de Red y Enlace (3, 2)

Se ocupan del direccionamiento lógico y físico.

  • 3. Red: Determina la mejor ruta para los paquetes usando **Direcciones IP**. Dispositivo clave: **Router**.
  • 2. Enlace de Datos: Gestiona el acceso al medio y el direccionamiento físico usando **Direcciones MAC**. Dispositivo clave: **Switch**.

Capa Física (1)

Maneja la transmisión binaria de datos.

  • 1. Física: Transmite bits (0s y 1s) a través del medio físico (cables o radiofrecuencia). Se ocupa de voltajes y tiempos.

La PDU: El Nombre del Dato en Cada Capa

A medida que el dato viaja de una capa a otra, se encapsula y cambia de nombre. Estos nombres se conocen como **Unidades de Datos de Protocolo (PDU):**

Capas 7, 6, 5 (Aplicación, Presentación, Sesión)

El dato se llama: **Dato (Data)**.

Capa 4 (Transporte)

El dato se llama: **Segmento** (con TCP) o **Datagrama** (con UDP).

Capa 3 (Red)

El dato se llama: **Paquete (Packet)**. Aquí se añade la Dirección IP.

Capa 2 (Enlace de Datos)

El dato se llama: **Trama (Frame)**. Aquí se añade la Dirección MAC.

Capa 1 (Física)

El dato se llama: **Bit** (0s y 1s).
