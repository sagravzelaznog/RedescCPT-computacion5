Configuración de Router con NAT, DHCP y Rutas Estáticas

1. Escenario de Práctica: Red de Dos Sucursales

Debes configurar el Router principal de una sucursal para que pueda comunicarse con una segunda sucursal y proveer servicios a su LAN.

Requisitos de la Sucursal Principal (R1)

  • IP Interna (LAN): **192.168.10.1 /24**.
  • Servicio **DHCP** para clientes.
  • Configuración de **NAT** para Internet (simulada).

Requisitos de Conectividad WAN

  • R1 conectado a Internet (simulado).
  • R1 debe conocer la red de la Sucursal B: **192.168.20.0 /24**.
  • Se debe crear una **Ruta Estática** para alcanzar la Sucursal B.

2. Pasos de Implementación en Packet Tracer

Esta práctica integra NAT, DHCP y el enrutamiento manual:

Configuración de Servicios (R1)

  1. Asigna la IP 192.168.10.1 al Router.
  2. Habilita DHCP en R1 para la LAN 192.168.10.x.
  3. Configura la Interfaz WAN y simula el NAT hacia Internet (marcando la interfaz como *outside*).

Implementación de Ruta Estática

En R1, el comando de ruta estática para alcanzar la Sucursal B (192.168.20.0/24) debe ser configurado, apuntando al siguiente salto (la IP del Router 2).

3. Tarea: Prueba de Conectividad Integral

Tarea Práctica (Diagnóstico Final):

  1. Verifica que los PCs de la Sucursal 1 reciban IP por DHCP.
  2. Verifica que los PCs de la Sucursal 1 puedan hacer **ping** a un PC de la Sucursal 2.
