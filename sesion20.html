Semana 5, Sesión 20: El Router como Servidor DHCP 🔄

Definiendo el Pool de Direcciones y Excluyendo IPs

1. Habilitar y Deshabilitar el Servidor DHCP

En routers domésticos, el servidor DHCP está generalmente habilitado por defecto. Sin embargo, en redes empresariales o cuando se usan varios puntos de acceso, a veces es necesario **deshabilitarlo** para evitar conflictos (dos dispositivos repartiendo IPs) o centralizar el control.

Un error común de red: **Conflicto de DHCP**. Ocurre cuando dos routers/servidores en la misma LAN intentan repartir IPs, causando inestabilidad.

2. Configuración del Pool de Direcciones

El **Pool de Direcciones** es el rango de IPs que el router tiene permitido repartir a los clientes. Es crucial definirlo correctamente para evitar asignar IPs a dispositivos que ya usan IP Estática.

IP Inicial y Final

Definen los límites del rango a repartir (ej. 192.168.1.100 hasta 192.168.1.254).

Direcciones Reservadas/Excluidas

Son IPs que están dentro del rango de la red, pero que el servidor DHCP NO repartirá. Se usan para **asignaciones estáticas** críticas (servidores, impresoras).

3. Práctica: Configuración y Prueba en Packet Tracer

Tarea Práctica (Simulación - Continuación de S3 E11):

  1. En Packet Tracer, verifica que tu Router tenga el servicio DHCP habilitado.
  2. Modifica el Pool para que solo reparta IPs en el rango **192.168.1.50 a 192.168.1.100**.
  3. Conecta 3 PCs al Switch/Router y verifica que obtengan IPs dentro de ese nuevo Pool.
  4. Asigna manualmente la IP **192.168.1.10** al PC4 (IP Estática). ¿Debería funcionar el DHCP en PC4? (No, porque la configuración manual anula el DHCP).
