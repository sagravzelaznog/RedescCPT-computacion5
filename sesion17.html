Semana 5, Sesión 17: Configuración de Dispositivos Finales (IP Estática y Dinámica)

Conceptos Básicos y Fundamentos Teóricos

1. Elementos Esenciales de la Configuración IP

Cada dispositivo (Host) en una red necesita obligatoriamente cuatro piezas de información para comunicarse eficazmente:

Dirección IP

La identidad lógica única del dispositivo (ej. 192.168.1.10).

Máscara de Subred

Define los límites de la red local (ej. 255.255.255.0 o /24).

Puerta de Enlace (Gateway)

La dirección IP del router local. Es el punto de salida hacia otras redes (ej. 192.168.1.1).

Servidor DNS

El servidor que traduce nombres de dominio a IPs (ej. 8.8.8.8).

2. IP Dinámica (DHCP) vs. IP Estática

La forma en que asignamos la dirección IP tiene consecuencias en la administración de la red.

Asignación Dinámica (DHCP) - La Norma

La IP es asignada **automáticamente** por un servidor (usualmente el router). Es ideal para la mayoría de los dispositivos (laptops, móviles) ya que es fácil y no requiere intervención manual.

Asignación Estática - La Excepción

La IP es configurada **manualmente** y no cambia. Es obligatorio para dispositivos que necesitan ser encontrados por otros en la red (ej. **Servidores, Impresoras de Red, Routers**).

3. Software Gratuito: NetSetMan

**NetSetMan** es una herramienta gratuita y muy útil para técnicos que trabajan en diferentes redes (ej. en casa tienen una red y en la escuela otra). Permite guardar perfiles de configuración IP, Wi-Fi, DNS, etc., y cambiarlos con un solo clic.

Tarea Práctica (Windows):

  1. Instala NetSetMan (versión gratuita para uso no comercial).
  2. Crea un perfil llamado "Casa" con la configuración IP de tu hogar (dinámica o estática).
  3. Crea un perfil llamado "Taller" con una configuración estática ficticia (ej. 10.0.0.10, 255.0.0.0, Gateway 10.0.0.1).
