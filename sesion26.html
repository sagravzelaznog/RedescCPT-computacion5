Semana 7, Sesión 26: Troubleshooting (Fase II) 🛑

Direcciones IP Duplicadas, Fallos de Gateway y DNS

1. Errores Comunes de Configuración IP

La mayoría de los fallos de conectividad se deben a un error en la configuración manual o automática de la IP.

Dirección IP Duplicada

Dos dispositivos tienen la misma IP. El sistema operativo suele alertar, y la red se vuelve inestable para ambos hosts. Solución: Asegurarse de que el servidor DHCP excluya las IP estáticas.

Máscara Incorrecta

Si la máscara es incorrecta, el PC creerá que otros dispositivos están en su red cuando no lo están (o viceversa), impidiendo la comunicación local.

2. Fallos del Gateway y Problemas de Capa 3

Si el **Paso 2 de PING** funciona (llega al Gateway), pero el **Paso 3** falla (no llega a 8.8.8.8), el problem está en el router o en la conexión a Internet.

El Router no Enruta (Capa 3)

El router puede estar encendido, pero ha perdido la IP de su interfaz WAN (la que da al ISP) o su tabla de enrutamiento es incorrecta. Solución: Reiniciar el router o verificar la configuración WAN.

Error de DNS (Falla la Capa de Aplicación)

Si `ping 8.8.8.8` funciona pero `ping google.com` falla, el problema es que el PC no sabe cómo traducir el nombre. Solución: Verificar que la IP del servidor DNS en el PC y el Router sea correcta (ej. 8.8.8.8 o 1.1.1.1).

3. Tarea Práctica: Simulación de Falla Compleja

Escenario Práctico (Simulación en Packet Tracer):

  1. Crea una red con 1 Router, 1 PC. Asigna IP estática 192.168.1.10 y Gateway 192.168.1.1.
  2. Simula una **IP duplicada** asignando intencionalmente 192.168.1.10 a la interfaz LAN del Router.
  3. Intenta hacer ping desde el PC al Router. Observa el fallo y la inestabilidad.
  4. Corrige el fallo, cambiando la IP del Router a 192.168.1.1.
