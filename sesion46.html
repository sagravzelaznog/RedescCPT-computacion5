El Escaneo de Puertos como Herramienta de Auditoría Ética

1. Escaneo de Puertos: Conocer las Puertas Abiertas

El **Escaneo de Puertos** es el proceso de sondear un host para determinar qué puertos TCP/UDP están **abiertos** (aceptando conexiones), **cerrados** (no hay servicio) o **filtrados** (bloqueados por un firewall).

Propósito Ético (Técnico)

Un técnico lo usa para auditoría: verificar que los únicos puertos abiertos sean los intencionales (ej. 80 para web, 22 para SSH) y que todos los demás estén cerrados o filtrados.

Propósito Malicioso

Los atacantes lo usan para encontrar un "punto débil" (un puerto abierto no asegurado) en el host objetivo antes de lanzar un ataque más profundo.

2. Nmap / Zenmap: El Estándar de la Industria

**Nmap** (Network Mapper) es el escáner de puertos de código abierto más potente y popular. **Zenmap** es simplemente la interfaz gráfica (GUI) de Nmap, haciéndolo más fácil de usar para principiantes.

Funcionalidades Clave

  • **Host Discovery:** Encuentra dispositivos activos en una red.
  • **Port Scanning:** Identifica puertos abiertos (ej. `-p 1-1000` para los primeros 1000 puertos).
  • **OS Detection:** Intenta adivinar el sistema operativo del host.

Simulación de Detección de Intrusos

Al escanear tu propia red con Zenmap, si encuentras un puerto abierto inesperado (ej. Telnet 23), has detectado una vulnerabilidad que debe corregirse de inmediato.

3. Tarea Práctica: Escaneo Básico de Puertos con Zenmap

Tarea Práctica (Software Gratuito):

  1. Descarga e instala **Zenmap**.
  2. Escanea la dirección de tu Gateway/Router (ej. `192.168.1.1`).
  3. Usa un escaneo de tipo **"Intense scan"** (o escribe `-p 1-1000` para enfocarte en los puertos comunes).
  4. Documenta los puertos que el router tiene abiertos (típicamente 80, 443 para la interfaz web, o 23/22 si tiene Telnet/SSH habilitado).
