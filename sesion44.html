Configuración de un Servidor Básico y Uso de Herramientas Linux

1. Escenario de Práctica: Configuración de un Servidor Web Básico

Se te pide configurar un servidor Linux básico (simulado en Packet Tracer o tu VM/WSL) para alojar una página web simple y verificar su estado.

Configuración del Servidor

  • Sistema Operativo: Linux (simulado).
  • IP Estática: **192.168.1.50 /24**.
  • Servicio Web (HTTP/Port 80) activo.

Tareas de Verificación

  • Acceso remoto mediante SSH (simulado con PuTTY).
  • Verificación de puertos abiertos con **`ss`** o **Zenmap**.
  • Comprobación de latencia desde el cliente (`ping`).

2. Pasos de Implementación y Diagnóstico

Esta práctica utiliza todas las herramientas aprendidas hasta ahora en un entorno de servidor real:

Configuración (Simulada)

  1. Asegura la IP estática en el servidor (192.168.1.50).
  2. Verifica que el servicio SSH esté activo (simulado en el servidor).
  3. En el cliente, usa **PuTTY** para conectarte (simulado).

Diagnóstico con Zenmap (Cliente)

Desde la PC cliente, usa Zenmap (o Advanced IP Scanner) para escanear la IP del servidor (192.168.1.50). La herramienta debe reportar que el **Puerto 80 (HTTP)** y el **Puerto 22 (SSH)** están **abiertos**.

3. Tarea: Escaneo y Verificación de Puertos

Tarea Práctica (Software Gratuito):

  1. Abre **Zenmap** (o Nmap) y escanea una IP conocida de tu red (ej. la de tu router o una VM).
  2. Identifica si el Puerto 22 (SSH) o el Puerto 23 (Telnet) están abiertos.
  3. Documenta el estado de esos puertos. Si Telnet está abierto, es un riesgo de seguridad que debe cerrarse.
