Semana 14, Sesión 54: Documentación Escrita 📋

Elaboración de la Tabla de Direccionamiento y Registros de Seguridad

1. La Tabla de Direccionamiento (El Registro de Inventario)

La tabla de direccionamiento es la documentación formal y detallada que muestra el estado de cada dispositivo que tiene una IP fija. Esto es crucial para la referencia rápida del técnico.

Elementos Mínimos de la Tabla:

  • Nombre del Dispositivo (ej. SRV-ARCHIVOS).
  • IP Asignada (ej. 192.168.10.2).
  • Máscara (ej. 255.255.255.0).
  • Gateway (ej. 192.168.10.1).
  • MAC Address (del dispositivo físico).
  • Función (Servidor, Gateway, Cliente, etc.).

2. Registros de Servicios y Seguridad

Además de las IPs, la documentación profesional incluye los registros de configuración de los servicios implementados:

Registro DHCP

Documentar el **Pool inicial** y **Pool final** del DHCP, el **tiempo de concesión** y cualquier **IP excluida** (las IPs estáticas).

Registro de Credenciales

Registrar las contraseñas cifradas del Router (SSH) y la clave Wi-Fi (WPA2/WPA3). **IMPORTANTE:** Estas contraseñas deben estar seguras y entregadas de forma responsable.

Registros de Puertos

Documentar si se habilitó algún **Port Forwarding** (ej. Puerto 80 redirigido al Servidor). Esto es una referencia vital para la seguridad futura.

3. Tarea: Completar los Registros Escritos

Tarea Práctica (Documentación):

  1. Crea la Tabla de Direccionamiento para el Router, Servidor e Impresora.
  2. Crea el registro de DHCP con la IP inicial y final del Pool.
  3. Documenta la configuración de seguridad del Router (SSH habilitado/Telnet deshabilitado).
