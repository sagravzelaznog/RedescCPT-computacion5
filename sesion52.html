Configuración del Router como Servidor DHCP y Acceso SSH

1. Configuración del Servicio DHCP en el Router

Ahora que los dispositivos estáticos están configurados, activaremos el **DHCP** para que los 20 PCs de los empleados obtengan su IP automáticamente.

El Pool de Direcciones

Configura el rango DHCP para que inicie después de la última IP estática (192.168.10.3) y use el Pool de **192.168.10.50 a 192.168.10.75**.

Verificación de Clientes

En todos los PCs, cambia la configuración de IP de "Estática" a **"DHCP"**. Verifica que reciban una IP dentro del rango del Pool (entre .50 y .75).

2. Implementación de SSH (Acceso Remoto Seguro)

Para la administración remota, el Router debe ser accedido de forma segura. Usaremos **SSH** y deshabilitaremos el inseguro Telnet.

Pasos Clave (CLI en Router)

  1. Asigna un nombre de dominio (`ip domain-name tech.com`).
  2. Genera las claves de cifrado (`crypto key generate rsa`).
  3. Configura usuarios y aplica el transporte SSH a las líneas VTY (`transport input ssh`).

Prueba de SSH

Desde un PC (en el *Desktop* del PC en Packet Tracer), usa el cliente **SSH** para intentar conectarte a la IP del Router (192.168.10.1). Debe pedirte usuario y contraseña y mostrar la línea de comandos del Router.

3. Tarea: Configuración Final y Backup

Tarea Práctica (Configuración Final):

  1. Habilita el servicio DHCP en el Router con el Pool correcto.
  2. Verifica que los PCs reciban IPs del Pool.
  3. Configura y prueba el acceso **SSH**.
  4. ¡Guarda la configuración! (**`copy running-config startup-config`**).
