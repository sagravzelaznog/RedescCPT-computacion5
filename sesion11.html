Semana 3, Sesión 11: DHCP y DNS 🤖

Automatización de IP y Resolución de Nombres de Dominio

1. DHCP: Protocolo de Configuración Dinámica de Host

El **DHCP (Dynamic Host Configuration Protocol)** permite que un servidor asigne automáticamente una dirección IP, máscara de subred, puerta de enlace y servidor DNS a cualquier dispositivo que se conecte a la red. Esto evita la configuración manual y reduce errores.

El Proceso DORA (Descubrimiento)

Es el ciclo de cuatro pasos que un cliente sigue para obtener una IP:

  1. **Discover (Descubrir):** El cliente busca un servidor DHCP.
  2. **Offer (Oferta):** El servidor le ofrece una IP disponible.
  3. **Request (Solicitud):** El cliente solicita formalmente esa IP.
  4. **Acknowledge (Aceptación):** El servidor confirma y otorga la IP por un tiempo (tiempo de concesión o *lease time*).

Componentes Clave

  • **Pool de Direcciones:** El rango de IPs que el servidor puede repartir.
  • **Tiempo de Concesión:** El tiempo que el cliente puede usar la IP antes de solicitar renovarla.
  • **Direcciones Excluidas:** IPs reservadas (ej. para el router o servidores) que no se repartirán.

2. DNS: Sistema de Nombres de Dominio

El **DNS (Domain Name System)** es el servicio que traduce nombres de dominio (ej. `google.com`) a direcciones IP numéricas (ej. `142.250.70.196`). Sin DNS, tendríamos que memorizar IPs para cada sitio web.

El Proceso de Resolución

Cuando escribes un dominio, tu PC pregunta a un servidor DNS. Si este no lo sabe, pregunta a los servidores **raíz**, luego a los servidores **TLD** (.com, .org) y finalmente al servidor **autoritativo** del dominio, hasta encontrar la IP correcta.

Comando Práctico: NSLOOKUP

Este comando nos permite consultar directamente a un servidor DNS para obtener la IP de un dominio.

3. Práctica: Configuración de DHCP en Packet Tracer

En esta práctica, implementaremos un servidor DHCP en un router (o servidor dedicado) para automatizar la configuración de dos PCs.

Tarea Práctica (Simulación):

  1. Crea un Router, un Switch y dos PCs en Packet Tracer.
  2. Configura la interfaz del Router (Gateway) con una IP estática (ej. 192.168.1.1).
  3. Habilita el servicio DHCP en el Router, definiendo un Pool que reparta de 192.168.1.100 a 192.168.1.200.
  4. En los PCs, cambia la configuración de IP de "Estática" a **"DHCP"**.
  5. Verifica que los PCs reciban automáticamente la IP del Pool (ej. 192.168.1.100).
