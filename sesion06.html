Semana 2, Sesión 6: El Modelo TCP/IP 🌐

El Estándar Real de Internet y el Proceso de Encapsulamiento

1. El Modelo TCP/IP (El Estándar Práctico)

Aunque el Modelo OSI es excelente para la enseñanza, el modelo que realmente utiliza Internet es el **Modelo TCP/IP**. Este modelo es más sencillo, ya que consolida algunas de las siete capas del OSI en solo cuatro capas:

4. Aplicación

Agrupa las capas 5, 6 y 7 de OSI (Aplicación, Presentación, Sesión). Contiene protocolos como **HTTP, DNS**.

3. Transporte

Similar a la Capa 4 de OSI. Protocolos clave: **TCP** (confiable) y **UDP** (rápido).

2. Internet

Similar a la Capa 3 de OSI (Red). Se encarga del direccionamiento lógico usando **IP** y del enrutamiento.

1. Acceso a Red

Agrupa las capas 1 y 2 de OSI (Física y Enlace de Datos). Maneja la transmisión de bits y las **Direcciones MAC**.

2. El Proceso de Encapsulamiento 📦

Cuando envías un mensaje, este no se manda completo. El **encapsulamiento** es el proceso mediante el cual los datos bajan por las capas del modelo TCP/IP (o OSI) y se les añade información de control (encabezados o *headers*) en cada nivel. Esto es como meter una carta en un sobre tras otro.

Capas Superiores (Datos)

Tu mensaje original se convierte en **Datos** y baja a la Capa de Transporte.

Capa de Transporte (Segmento/Datagrama)

Se añade el encabezado TCP/UDP (que incluye los **Puertos**). El paquete se llama **Segmento** (TCP) o **Datagrama** (UDP).

Capa de Internet (Paquete)

Se añade el encabezado IP (que incluye las **Direcciones IP** de origen y destino). Ahora es un **Paquete**.

Capa de Acceso a Red (Trama)

Se añade el encabezado de Enlace (que incluye las **Direcciones MAC**). Ahora es una **Trama** y está lista para convertirse en bits y viajar por el medio físico.

Práctica: Visualizando el Encapsulamiento en Packet Tracer

Usaremos el modo de simulación de Packet Tracer para ver este proceso en acción. Es la mejor forma de entender cómo se añaden los encabezados.

Tarea Práctica (Simulación):

  1. Abre el Packet Tracer y crea una red simple (PC conectado a un Switch, Switch conectado a otro PC).
  2. Cambia al **Modo Simulación** (el botón de reloj).
  3. Haz que el PC1 envíe un paquete **Ping** al PC2.
  4. Haz clic en el paquete mientras se mueve y revisa las pestañas "Inbound PDU" y "Outbound PDU".

Observa cómo el paquete comienza con datos (Capas TCP/IP 4), luego recibe encabezados IP (Capa 3) y finalmente encabezados Ethernet (Capa 2).
