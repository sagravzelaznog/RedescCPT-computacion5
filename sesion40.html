Implementación de SSH, WPA3 y Pruebas de Acceso Remoto

1. Escenario de Práctica: Securización de un Router

Se te pide configurar un router y un servidor de forma segura para soportar teletrabajo y evitar accesos no autorizados a la red Wi-Fi.

{{ ... }}

Requisitos de Seguridad Inalámbrica

  • Usar cifrado **WPA2-AES** (simulando WPA3 si el dispositivo no lo soporta).
  • Configurar una lista de **Filtrado MAC** para solo permitir el acceso a tu PC.

Requisitos de Acceso Remoto

  • Habilitar **SSH** en el Router (configuración de línea de comandos).
  • **Deshabilitar Telnet** en el Router.
  • Simular la conexión al Router usando **PuTTY** (cliente SSH).

2. Pasos de Implementación en Packet Tracer (Configuración CLI)

Esta práctica requiere usar la interfaz de línea de comandos (CLI) del Router en Packet Tracer:

Configuración SSH (CLI)

  1. Configura el nombre del dominio (`ip domain-name ejemplo.com`).
  2. Genera claves RSA (`crypto key generate rsa`).
  3. Activa SSH y configura el usuario (`line vty 0 4`, `transport input ssh`).

Prueba de Seguridad

Intenta conectarte al Router por **Telnet** (debe fallar). Luego, conéctate por **SSH** (debe ser exitoso).
